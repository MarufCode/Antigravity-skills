# Test Plan Creator Skill

A specialized skill for generating high-impact, 1-page test plans that follow industry standards.

## Overview

Software testing requires clear direction. This skill automates the creation of professional test plans, ensuring no critical section (like Exit Criteria or Risks) is forgotten. It uses a curated template designed to satisfy both project stakeholders and technical interviewers.

## Key Features

- **Standardized Template**: Uses the high-praise "1-page Test Plan" structure.
- **Scoping Assistance**: Helps define what's in and out of scope based on your app type.
- **Risk Evaluation**: Includes placeholders for common project risks and mitigation plans.
- **Markdown Ready**: Outputs formatted text ready for Jira, Confluence, or GitHub.

## How to Trigger

- "Create a test plan for [App Name]"
- "Generate a QA document using the 1-page template"
- "Draft a release test plan"

## The Template Structure

1. **Objective**: Release readiness.
2. **Scope**: What is (and isn't) being tested.
3. **Approach**: Manual vs Automation strategy.
4. **Environment**: Tools, browsers, and test labs.
5. **Entry & Exit**: When to start and when to stop.
6. **Deliverables**: Reports and test cases.
7. **Risks**: Potential blockers.
8. **Approval**: Authority sign-off.
